# P8.5 流式结果交付设计（Batch B · Tier 1）

> 日期：2026-04-20
> 范围：`.claude/worktrees/si-search-optimization`
> 背景：R5 S7 → R6 P8.5 backlog（"results[] 仍 O(匹配数)"）
> 前置：Batch A 完成（P7.3/P7.4/P7.5 + S6 AbortSignal 已到位，为流式取消支撑）

---

## 一、目标

把"搜索 → 返回全量 `SearchResult[]`"改造为"搜索 → 按批流式向 webview 推送"。核心收益：

- 匹配量大（万级/百万级）时 webview 不卡
- 首屏更快（第一批结果到达即渲染）
- 内存峰值不再 O(匹配数)

---

## 二、现状问题

**当前链路**（`messageRouter.ts:38-` 简化版）：
```ts
const results = await executeSearchWithIndex(...)  // 等全部完成
webview.postMessage({ command: 'searchResults', results })  // 一次性推
```

**瓶颈**：
- `executeSearch` 内部已是流式 readline（P7.4），但对外仍 `Promise<SearchResult[]>` —— 把流式硬凝固
- `postMessage` 单条消息带 N 万条 result JSON —— webview JS 主线程卡顿
- Webview UI 一次性 render N 万 DOM —— 卡死

---

## 三、设计决策

### 3.1 API 形状

**核心**：把 `executeSearch` 从 `Promise<SearchResult[]>` 改为 `AsyncIterable<SearchResult>`（或 `AsyncIterable<SearchResult[]>` 批）。

| 选项 | 优点 | 缺点 |
|---|---|---|
| `AsyncIterable<SearchResult>` 单条 | 最细粒度，for await 友好 | 每条都 yield 开销大 |
| **`AsyncIterable<SearchResult[]>` 按批** | 适配 postMessage 批量 | 需要定批策略 |
| 回调 `onBatch(results)` | 最简单 | 不够声明式 |

**选 `AsyncIterable<SearchResult[]>` 按批**。批大小默认 100，可配。

### 3.2 搜索引擎改造

`searchEngine.ts` 现状：`Promise<SearchResult[]>` 累计 results[]。改造：

```ts
export async function* executeSearchStream(
    query: string, workspaceRoot: string, options: SearchOptions,
    includeExtensions: string[], excludePatterns: string[],
    signal?: AbortSignal,
    batchSize = 100,
): AsyncGenerator<SearchResult[]> {
    // ... 保留 spawn + readline + abort 链路
    let buf: SearchResult[] = [];
    rl.on('line', line => {
        const parsed = parseRgLine(line, workspaceRoot);
        if (parsed) { buf.push(parsed); if (buf.length >= batchSize) flush(); }
    });
    // close 时 yield 剩余 buf;error/abort 时 throw
}
```

关键挑战：`readline` 的 `on('line')` 是推模型，`async function*` 是拉模型。需要用 **背压 bridge**：

```ts
// 伪代码:用 queue + waiter 把 event emitter 接到 async iterator
class Bridge<T> {
    private queue: T[] = [];
    private waiter?: (v: IteratorResult<T>) => void;
    private done = false;
    private err?: Error;
    push(item: T) { /* enqueue or resolve waiter */ }
    close() { /* set done, resolve waiter if any */ }
    fail(e: Error) { /* set err */ }
    next(): Promise<IteratorResult<T>> { /* return queued or await */ }
}
```

这是 P8.5 的**核心技术风险点**，需实现+测试。

保留现有 `executeSearch` 作为 Promise 薄包装供不需要流式的调用方（如 index fallback）。

### 3.3 messageRouter 改造

```ts
case 'search': {
    let batchIdx = 0;
    try {
        for await (const batch of executeSearchStream(...)) {
            resultsPanel.postMessage({ command: 'appendResults', batchIdx, results: batch });
            batchIdx++;
        }
        resultsPanel.postMessage({ command: 'searchComplete', total: batchIdx });
    } catch (err) {
        if (err.name === 'AbortError') {
            resultsPanel.postMessage({ command: 'searchAborted' });
        } else {
            resultsPanel.postMessage({ command: 'searchError', message: err.message });
        }
    }
}
```

### 3.4 webview 改造

Panel 侧消息 handler 从 "replace" 变 "append"：

```ts
case 'appendResults': {
    if (msg.batchIdx === 0) { clearResults(); }
    appendResults(msg.results);  // 增量 DOM
    break;
}
case 'searchComplete': finalizeUI(msg.total); break;
case 'searchAborted':  showAbortedBanner(); break;
```

**UI 增量渲染策略**：
- DocumentFragment 批量 append
- 超过 N 千条时启用虚拟滚动（独立工单）

### 3.5 SymbolIndex fallback 路径

`executeSearchWithIndex` 先查 SymbolIndex（已是 in-memory 同步）。符号 hit 时：
- 当前返回 `SearchResult[]`
- 流式模式下：`executeSearchStream` 先 yield 一批符号结果，再决定是否 fallback 到 rg 流

**简化方案**：SymbolIndex hit 时不走流式（符号量通常 < 数千，不是瓶颈）。仅 rg 路径流式。

## 四、取消语义

S6 AbortSignal 已到位。流式模式下：
- 调用方 `ctrl.abort()` → `executeSearchStream` throw `AbortError`
- `messageRouter` catch 后发 `searchAborted`
- 已投递的 batch 保留在 UI（用户能看到部分结果）
- 新搜索覆盖旧（webview append 前 clearResults）

## 六、风险

| 风险 | 等级 | 缓解 |
|---|---|---|
| Bridge 背压 bug（memory leak 或死锁）| HIGH | Bridge 独立单测 + 压力测试 |
| Webview IPC 频率过高（每批一次 postMessage）| MED | batchSize 调优，默认 100 |
| 取消后孤儿进程 | LOW | S6 已保证 SIGTERM + rl.close |
| 向后兼容（外部调用方期望 Promise）| LOW | 保留 `executeSearch` 薄包装 |
